# ApiaryLens 0.1.0-rc.6

## Release Status

This is the corrective MVP acceptance candidate, not a stable release. It supersedes
0.1.0-rc.5 after exact published-byte GCP cleanup proved that a retry from rc.4's
partial uninstall state could not parse Compose because rc.4 had already removed the
secret files. Do not deploy rc.5; it remains published only as an immutable audit
record.

## Changes from rc.5

- Recreates missing secret-file mount points with empty, mode-restricted placeholders
  only while completing a full uninstall from a partially removed legacy target.
- Requires Docker Compose removal to succeed before deleting the validated target.
- Removes the complete target, including releases, secrets, and backups, after the
  Compose resources are gone.
- Preserves the recoverable keep-data path and all published product contracts.
- Adds regression coverage for partial legacy cleanup and complete target removal.

## Included MVP

- Installable offline-capable React PWA with IndexedDB drafts, staged photos,
  synchronization, conflicts, and safe updates
- Built-in owner, beekeeper, and viewer identity with protected first-owner claim,
  invitations, recovery, and server-side authorization
- Family-to-professional apiary records, private media, history, follow-ups, export,
  backup, and restore
- Cloudflare Worker/D1/R2 and portable Node/SQLite/filesystem Compose profiles
- Scout Bee guided install, update, backup, restore, diagnostics, and uninstall
- Public .org, .app, and .dev frontends with synchronized documentation and support

## Remaining Acceptance Gates

## Upgrade and Recovery

Back up before updating. rc.6 introduces no data migration and preserves API 1.0,
sync 1, migration 0004, deployment plan 1, export 1, and local store 1. Verify the
content-addressed artifact and repository attestation before applying it.

## Support Window

Release candidates receive best-effort support until replaced by a newer candidate
or the first stable release. Report security defects and data-loss risks privately
according to SECURITY.md.
