# ApiaryLens 0.1.0-rc.4

## Release Status

This is the corrective MVP acceptance candidate, not a stable release. It supersedes 0.1.0-rc.3, whose exact published Cloudflare bytes passed but whose Scout Bee Compose path could not create the documented default /opt/apiarylens directory on a clean Ubuntu host. Do not deploy rc.3; it remains published only as an immutable audit record.

## Changes from rc.3

- Adds an explicit remote install-folder access phase to Compose preflight.
- Safely creates a missing target only when the user has passwordless sudo.
- Rejects symlink, root, non-normalized, unwritable, or foreign-owned install targets.
- Adds unit coverage for valid, unsafe, and inaccessible Compose targets.
- Preserves the API, sync, migration 0004, export, local-store, and deployment-plan contracts.

## Included MVP

- Installable offline-capable React PWA with IndexedDB drafts, staged photos, synchronization, conflicts, and safe updates
- Built-in owner, beekeeper, and viewer identity with protected first-owner claim, invitations, recovery, and server-side authorization
- Family-to-professional apiary records, private media, history, follow-ups, export, backup, and restore
- Cloudflare Worker/D1/R2 and portable Node/SQLite/filesystem Compose profiles
- Scout Bee guided install, update, backup, restore, diagnostics, and uninstall
- Public .org, .app, and .dev frontends with synchronized documentation and support

## Remaining Acceptance Gates

- Full rc.4 local verification, public CI, signing, publication, and independent attestation verification
- Exact downloaded rc.4 Cloudflare and clean Compose deployment smoke
- Physical iPhone, iPad, and computer offline/relaunch/synchronization journey
- Separate-device invitation acceptance and pending-work-across-update verification
- Manual keyboard, NVDA, VoiceOver, 200% zoom, and forced-colors review
- Final project-owner product, visual, documentation, and stable-release acceptance

## Upgrade and Recovery

Back up before updating. rc.4 keeps API 1.0, sync 1, database migration 0004, deployment plan 1, export 1, and local store 1; no data migration is introduced. Verify the content-addressed artifact and repository attestation before applying it.

## Support Window

Release candidates receive best-effort support until replaced by a newer candidate or the first stable release. Report security defects and data-loss risks privately according to SECURITY.md.
