# ApiaryLens 0.1.0-rc.5

## Release Status

This is the corrective MVP acceptance candidate, not a stable release. It supersedes
0.1.0-rc.4 after final GCP remove-data acceptance proved that Compose removed the
live Docker volume and secrets but retained verified backups under the installation
directory. Do not deploy rc.4; it remains published only as an immutable audit record.

## Changes from rc.4

- Makes full Compose uninstall remove the complete validated installation directory,
  including releases, secrets, and retained backup archives.
- Preserves the explicit keep-data path unchanged for recoverable reinstall.
- Adds regression coverage for full target removal.
- Preserves API 1.0, sync 1, migration 0004, export 1, local store 1, and deployment
  plan 1; no data migration is introduced.

## Included MVP

- Installable offline-capable React PWA with IndexedDB drafts, staged photos,
  synchronization, conflicts, and safe updates
- Built-in owner, beekeeper, and viewer identity with protected first-owner claim,
  invitations, recovery, and server-side authorization
- Family-to-professional apiary records, private media, history, follow-ups, export,
  backup, and restore
- Cloudflare Worker/D1/R2 and portable Node/SQLite/filesystem Compose profiles
- Scout Bee guided install, update, backup, restore, diagnostics, and uninstall
- Public .org, .app, and .dev frontends with synchronized documentation and support

## Remaining Acceptance Gates

- AWS Ubuntu Compose compatibility and cleanup evidence
- Physical iPhone and iPad offline/relaunch/synchronization journey
- Separate-device invitation acceptance and pending-work-across-update verification
- Manual keyboard, NVDA, VoiceOver, 200% zoom, and forced-colors review
- Final project-owner product, visual, documentation, and stable-release acceptance

See the [MVP UAT record](../testing/mvp-uat.md) for evidence.

## Upgrade and Recovery

Back up before updating. rc.5 keeps every published contract from rc.4 and introduces
no data migration. Verify the content-addressed artifact and repository attestation
before applying it.

## Support Window

Release candidates receive best-effort support until replaced by a newer candidate
or the first stable release. Report security defects and data-loss risks privately
according to SECURITY.md.
