# Authentication, Authorization, and Sharing

## Authentication

Start simple:

- Local username/password
- Optional magic links later
- Optional OAuth later
- Optional OIDC/Keycloak later

## Organization Model

Use organizations from the beginning so the app can support families, clubs, mentors, commercial teams, and future SaaS.

## Roles

- Owner
- Admin
- Apiary Manager
- Inspector
- Mentor
- Read-only Viewer
- Club Member

## Sharing

Support private sharing, family sharing, mentor sharing, bee club sharing, and public read-only reports.
